🔧 API ENDPOINTS CREATED (10)
═══════════════════════════════════════════════════════════════════════════

ALL UNDER: /api/v1/merchant/transactions/
WITH JWT AUTHENTICATION MIDDLEWARE

1. GET    /data               → Paginated transactions list
2. GET    /statistics         → Transaction statistics (sale/refund/void)
3. GET    /export             → Export to CSV with filters
4. POST   /bulk-delete        → Delete multiple transactions
5. GET    /{id}               → Single transaction details
6. POST   /{id}/void          → Void transaction
7. POST   /{id}/refund        → Refund transaction
8. POST   /{id}/send-receipt  → Send receipt via email
9. GET    /{id}/receipt       → View receipt

HELPER ENDPOINT:
10. GET   /api/softpos/merchant/terminals → Get terminals dropdown

✨ WHAT'S DIFFERENT FROM BLADE VERSION?
═══════════════════════════════════════════════════════════════════════════

BEFORE (Blade/jQuery):
❌ Full page reload on every action
❌ DataTables library with jQuery
❌ Separate JavaScript file with inline scripts
❌ Mixed server/client logic
❌ Hard to maintain and extend

AFTER (React):
✅ No page reloads - instant updates
✅ Modern React DataTable component
✅ Clean component architecture
✅ Clear separation of concerns
✅ Easy to maintain and extend
✅ Better performance
✅ Integrated with merchant-app.jsx router
✅ Shared UI components and state
