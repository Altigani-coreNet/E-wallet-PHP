🗺️ NAVIGATION FLOW
═══════════════════════════════════════════════════════════════════════════

USER JOURNEY (NO PAGE RELOADS!):

1. Visit: /merchant/transactions
   → React: MerchantTransactions component loads
   → Shows: Skeleton → Statistics → Table

2. Click: "View" button on any transaction
   → React: navigate('/merchant/transactions/123')
   → React Router switches component
   → React: TransactionDetail component loads
   → API: Fetches transaction data
   → Shows: Skeleton → Full transaction details

3. Click: Back button
   → React: navigate('/merchant/transactions')
   → React Router switches back
   → React: MerchantTransactions component
   → Shows: Previous state preserved!

BENEFITS:
✅ INSTANT navigation (no page reload!)
✅ Smooth transitions
✅ State preservation
✅ Professional SPA experience

📡 API ENDPOINTS
═══════════════════════════════════════════════════════════════════════════

ALL UNDER: /api/v1/merchant/transactions/

INDEX PAGE:
• GET  /data                → Paginated transactions list
• GET  /statistics          → Transaction statistics (sale/refund/void)
• GET  /export              → Export to CSV with filters

DETAIL PAGE:
• GET  /{id}                → Single transaction with all relationships

HELPER:
• GET  /api/softpos/merchant/terminals → Terminals dropdown

ALL REQUIRE: JWT Authentication via middleware
